Xerom is the Ether-1 side chain.

Xerom is our rapid development chain, whilst Ether-1 is the mother chain that gets all the stable/proven improvements.

How Xerom works with Ether-1

Xerom provides a different monetary policy as well as a different ledger for ethoFS hashes to be stored in. This allows for a more Decentralized ethoFS ecosystem as ethoFS is no longer reliant on a single chain.

Our Nodes

Xerom introduces nodes which earn a share of the block reward by hosting content on the ethoFS network as well as securing the Xerom chain by running a geth instance which verifies transactions on the Xerom Network.
Chain Node

5,000 XERO

  • OS: Linux
  • RAM: 1GB
  • CPU: Intel/ARM - 1 core minimum.
  • Storage: 16GB HDD/SSD/SD Card.
  • Public static IP address.
  • Chain Nodes play a very important part of the XERO ecosystem. Chain nodes run a XERO Geth instance, which means your Chain node helps to verify the legitimacy of transactions. Since the Geth instance is not extremely resource intense the Chain node has a very low resource requirement.
Xero Node

20,000 XERO

  • OS: Linux
  • RAM: 2GB RAM
  • CPU: Intel/ARM - 1 core minimum.
  • Storage: 40GB SSD.
  • Public Static IP address.
  • Xero Nodes are specialized nodes which run DHT ethoFS binaries as well as Geth instances. Xero Nodes help secure the chain by verifying transactions on the network as well as help speed up the ethoFS network by running a DHT binary which helps ethoFS connect users to content hosted on ethoFS quicker.
Link Node

40,000 XERO

  • OS: Linux
  • RAM: 4GB RAM
  • CPU: Intel/ARM - 2 cores recommended.
  • Storage: 80GB SSD.
  • Public Static IP address.
  • Link Nodes run ethoFS, Geth and IPFS binaries. Link Nodes help secure the chain by verifying transactions on the network as well as host content uploaded to the ethoFS network by running an ethoFS and IPFS binary.
Super Node

80,000 XERO

  • OS: Linux
  • RAM: 8GB RAM
  • CPU: Intel/ARM - 2 cores recommended.
  • Storage: 120GB SSD.
  • Public Static IP address.
  • Super Nodes are the brother to Link Nodes they run ethoFS, Geth and IPFS binaries. Super Nodes have an increased collateral requirement as well as an increase in the storage requirement - Super Nodes perform the same functions as a Link Node.
